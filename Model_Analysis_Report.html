Model Analysis Report: MNIST Digit Recognizer

Confusion Matrix (Validation Set)

This chart shows where the model is succeeding and where it is getting confused.

Type I and Type II Errors

Type I Error (False Positive)

A "false alarm." This is when the model incorrectly predicts a class.
Chart Example: Looking at the column for "Predicted Label = 4", the model predicted '4' when the true label was '9' (16 times). This is a Type I error (a false positive) for the '4' class.

Type II Error (False Negative)

A "miss." This is when the model *fails to predict* the correct class.
Chart Example: Looking at the row for "True Label = 9", the model failed to identify the '9' and instead predicted '4' (16 times). This is a Type II error (a miss) for the '9' class.

Strong Conclusions from the Chart

  1. The model is generally strong but has clear weaknesses. The high numbers on the diagonal (e.g., '0' correct 100 times, '1' correct 104 times) show it performs well on most digits.
  2. The model's single biggest problem is confusing '9' and '4'. The largest single error is predicting '4' when the true digit was '9' (16 times). The second-largest is predicting '9' when the true digit was '4' (10 times).
  3. The digit '2' is also problematic. The model struggles with '2', confusing it most often with '3' (9 times), but also scattering errors across '4', '6', '7', and '8'.
  4. '0' and '1' are the best-performing classes. The model has almost no trouble identifying '0' and '1', with very few false positives or false negatives.
